## 1. 개요·목표

### 1.1 문제 정의 (Pain + 실패 지표)

**타깃:**

- 자기계발 의지는 있지만,
    - 방해금지/집중모드를 쉽게 끄고
    - 유튜브·SNS로 새는 **의지력 고갈형 지식노동자/직장인**

**핵심 Pain**

1. **방해금지/집중모드 유지 실패**
    - 잠깐 의지를 내서 방해모드를 켰다가,
        
        5–10분만 지나면 해제하고 다른 앱(유튜브/SNS 등)으로 새는 패턴.
        
2. **앱 전환 과잉 & 습관 루틴 분산**
    - 명상 앱, 영어 앱, 운동 앱, 독서 타이머 앱이 **각각 따로** 있고
        
        “지금 뭘 할지”를 고르는 데서 이미 결정 피로 발생.
        
3. **습관 기록·트래킹 부재**
    - “이번 달 내가 뭘 얼마나 했는지” 체감만 있고,
        
        **객관적인 그래프/기록이 없어** 동기 유지가 어려움.
        

**실패 KPI(현재 상태 가설)**

- 집중 시간 블록(예: 30분) 중
    - 방해금지/집중모드 계획 시간 대비 실제 유지 시간: **50% 이하**
- 자기계발 루틴(명상/독서/운동/영어/영양제 등)
    - 주당 목표 달성률: **30% 이하**
- 하루 중 “목적 없는 앱 전환(집중블록 동안)” 횟수: **10회 이상**

---

### 1.2 목표(Desired Outcome 수치화)

**북극성 KPI**

- **의도된 자기계발 블록의 실제 실행률**
    - (알람으로 트리거된 블록 중, 실제로 어떤 자기계발 활동을 시작한 비율)
    - 베이스라인: 40% (가설) → **목표: 70% 이상**

**보조 KPI**

1. 방해모드 해제 후 “목적 없는 앱 전환” 횟수
    - 베이스라인: 블록당 5회 이상 → **목표: 2회 이하**
2. 주간 자기계발 루틴 달성률
    - 베이스라인: 30% → **목표: 60% 이상**
3. 4주차 Retention (주 3일 이상 앱 사용 비율)
    - 베이스라인: 20% (가설) → **목표: 50% 이상**

## 3. 사용자 스토리와 수용 기준(AC)

### 3.1 메인 사용자 스토리

**Story 1 – 집중 블록 시작 시 런처 역할**

> As a
> 
> 
> 의지력 고갈을 자주 겪는 직장인(준호)
> 
> I want
> 
> 알람/집중 블록이 시작됐을 때, 자동으로 자기계발 옵션만 모여 있는 화면이 열리고
> 
> so that
> 
> 다른 앱으로 새지 않고, 그 순간에 할 자기계발 활동 하나를 바로 선택해 실행하고 싶다.
> 

**AC 1.1 – 트리거 연동**

- **Given** 사용자가 앱에서 “자기계발 블록 시간(예: 22:00)”을 설정했고, 해당 시간에 휴대폰 알람/푸시가 울릴 때
- **When** 사용자가 알림을 탭하거나, 휴대폰 잠금을 해제하면
- **Then** 홈 화면 대신(또는 바로 다음에) **Focus Habit Launcher 메인 화면**이 자동으로 뜨고,
- **And** 최소 3개 이상의 사전 설정 자기계발 옵션(명상/독서/영어/운동/영양제)이 노출되어야 한다.

**AC 1.2 – 앱 전환 최소화**

- **Given** 사용자가 런처 화면에서 특정 활동(예: 독서)을 선택했을 때
- **When** 활동을 시작하기 위해 화면을 탭하면
- **Then** **3탭 이하**로 활동 시작(타이머 시작 or 관련 앱/툴 연결)이 가능해야 한다.

**AC 1.3 – 의사결정 시간 제한**

- **Given** 런처 화면이 열린 시점
- **When** 30초 이내에 어떠한 활동도 선택되지 않는다면
- **Then** “오늘은 쉬고 싶다면 NO를 눌러도 괜찮다” 메시지와 함께 **‘NO’ 버튼**이 강조되고,
- **And** NO를 누르면 자기계발 명언 카드가 즉시 보여야 한다.

---

**Story 2 – 활동 시간 기록 및 자동 트래킹**

> As a
> 
> 
> 자기계발 활동을 꾸준히 하고 싶은 사용자
> 
> I want
> 
> 각 활동에 사용한 시간이 자동으로 기록되고
> 
> so that
> 
> 한 달 동안 내가 실제로 무엇을 얼마나 했는지 그래프와 기록으로 확인하고 싶다.
> 

**AC 2.1 – 활동 타이머**

- **Given** 사용자가 런처에서 독서를 선택했을 때
- **When** “시작” 버튼을 누르면
- **Then** 초/분 단위로 시간을 측정하는 **활동 타이머**가 시작되고,
- **And** 사용자가 “종료”를 누르는 시점까지의 시간이 해당 활동의 수행 시간으로 기록되어야 한다.

**AC 2.2 – 자동 저장**

- **Given** 타이머가 1분 이상 작동한 뒤 종료되었을 때
- **When** 앱이 포그라운드에서 벗어나거나, 사용자가 홈 화면으로 나가더라도
- **Then** 해당 활동, 날짜, 소요 시간(분 단위)이 **자동으로 로그 DB에 저장**되어야 한다 (사용자 별).

**AC 2.3 – 월간 그래프**

- **Given** 사용자가 “기록/통계” 탭을 한 번 탭했을 때
- **When** 현재 월에 대한 활동 기록이 존재한다면
- **Then** 최소 다음 항목이 시각화된 그래프(바/라인 등)로 노출되어야 한다.
    - 활동별 총 시간 (예: 독서 320분, 명상 90분 등)
    - 주차별 활동 시간 추이

---

**Story 3 – NO 선택 시 자기계발 명언 제공**

> As a
> 
> 
> 너무 지쳐서 오늘은 아무 것도 하기 싫은 사용자
> 
> I want
> 
> NO를 눌렀을 때 나를 비난하지 않고,
> 
> so that
> 
> 짧은 자기계발 명언이나 인용구로 기분을 다독이며 내일 다시 시작할 수 있도록 하고 싶다.
> 

**AC 3.1 – 부드러운 NO 플로우**

- **Given** 런처 화면에서 “오늘은 안 할래요(=NO)” 버튼을 눌렀을 때
- **When** NO가 선택되면
- **Then** “그래도 괜찮다”는 톤의 자체 메시지 + 자기계발 명언이 카드 형태로 즉시 노출되어야 한다.

**AC 3.2 – 내일 재시도 트리거**

- **Given** 사용자가 NO를 눌러 활동을 스킵한 날
- **When** 다음 자기계발 블록이 시작되는 날/시간
- **Then** “어제는 쉬었으니, 오늘은 이 중 하나만 해볼까요?”와 같은 톤의 메시지로 런처가 뜨고,
- **And** 사용자가 부담 없이 다시 선택할 수 있어야 한다.

## 4. 기능 요구사항 (Functional – MSCW)

### 4.1 Must (v1 출시 필수)

1. **알람/블록 트리거 기반 런처 화면 오픈**
    - 앱 내에서 “자기계발 블록 시간” 설정 기능
    - 해당 시간에 OS 알람/푸시 연동 → 알람 탭 시 런처 화면 자동 오픈
2. **자기계발 활동 옵션 설정 & 선택 UI**
    - 기본 제공 옵션: 명상, 영양제, 독서, 영어, 운동
    - 순서·표시/숨김 커스터마이즈
3. **활동별 타이머 & 시간 기록**
    - 시작/종료 버튼
    - 1분 이상 수행 시 자동 기록 저장
4. **월간 기록/그래프**
    - 활동별 총 시간, 주차별 시간 추이 시각화
5. **NO 플로우 & 명언 카드**
    - NO 버튼
    - 자기계발 명언 랜덤 노출(초기에는 로컬 리스트로 시작)
6. **기본 계정/로컬 저장 (단일 디바이스 기준)**
    - v1은 단일 디바이스 로컬 DB(혹은 간단 계정) 기준

### 4.2 Should (v1.5 이후)

1. **외부 앱 연동 (선택한 활동과 관련 앱 자동 실행)**
    - 예: 독서 선택 시 리디/밀리, 영어 선택 시 특정 앱 실행 등
2. **주간 리포트 요약**
    - “이번 주, 독서 3회 90분, 운동 2회 60분” 같은 카드
3. **사용자 지정 자기계발 활동 추가**
    - 예: “블로그 쓰기”, “코딩 연습” 등 커스텀 활동

### 4.3 Could (향후)

1. **AI 기반 추천 (지금 상태에 맞는 활동 추천)**
2. **친구/코치와 기록 공유**
3. **다른 집중 앱/포모도로 앱과의 API 연동**

### 4.4 Won’t (이번 범위에서 제외)

- 멀티 디바이스 동기화(웹/태블릿/PC)
- 완전한 OS-level Focus Mode 제어(시스템 권한이 필요한 차단 기능)

## 5. 비기능 요구사항 (NFR)

- **성능**
    - 런처 화면 오픈: 푸시/알람 탭 후 화면 표시까지 **p95 ≤ 1.0초**
    - 통계 화면 로딩: **p95 ≤ 1.5초** (한 달 데이터 기준)
- **신뢰성**
    - 하루 기록 저장 실패율: **0.5% 이하**
    - 앱 크래시율: 월간 세션 기준 **1% 이하**
- **보안/프라이버시**
    - 수집 데이터: 활동명, 시간, 날짜, 간단 설정값
    - 민감 개인정보/위치/연락처 미수집
    - 기본 OS 보안 스토리지 사용
- **비용**
    - 초기 버전: 단일 디바이스 로컬/간단 백엔드로 인프라 비용 최소화
- **모니터링**
    - 주요 로그:
        - 블록 시작/완료 이벤트
        - 활동 선택/종료 이벤트
        - NO 선택 이벤트
    - 간단 대시보드:
        - DAU/WAU
        - 활성 블록 수
        - 활동 완료율

## 6. 데이터·인터페이스 개요

### 6.1 핵심 엔터티

1. **User**
    - user_id
    - 생성일, 기기 정보(플랫폼, OS 버전)
2. **HabitActivity**
    - activity_id (명상/영양제/독서/영어/운동/커스텀 등)
    - 이름, 아이콘, 색상, 활성 여부
3. **SessionLog**
    - session_id
    - user_id
    - activity_id
    - start_time, end_time, duration_min
    - source (런처에서 시작, 외부 앱에서 시작 등)
4. **BlockSchedule**
    - schedule_id
    - user_id
    - 시작 시간, 반복 패턴(매일, 평일 등)
    - 활성 여부
5. **Quote**
    - quote_id
    - 문구, 출처, 태그(동기/자기수용/성장 등)

### 6.2 외부/내부 인터페이스

- **OS 알람/푸시 연동**
    - iOS/Android Notification/Alarm Manager 사용
    - “알람 탭 → 특정 딥 링크로 앱 열기” 수준 (OS 제한 내)
- **(향후) 외부 앱 딥링크**
    - 활동별 딥링크 URL 설정 필드
    - 예: `read` 활동 → 리디 앱 특정 URI

## 7. 범위(In/Out), 리스크·가정·의존성

### 7.1 In Scope

- 모바일 앱(v1: Android 우선 가정)
- 알람/푸시 기반 런처 화면
- 활동 선택·타이머·기록·그래프
- NO 플로우 & 명언 카드

### 7.2 Out of Scope

- 시스템 전체를 잠그는 강력한 포커스 모드(앱 차단/화이트리스트)
- 멀티 디바이스(웹/태블릿) 동기화
- 소셜 기능(친구·코치 공유)

### 7.3 리스크

1. **OS 권한 제한**
    - iOS/Android 모두 “잠금 해제 시 강제 앱 실행”에 제약이 있으므로,
        
        현실적으로는 “알람 탭 → 앱 실행” 수준에 머물 가능성.
        
2. **의지력 vs 강제력 균형**
    - 너무 강하게 앱을 띄우거나 다른 앱 사용을 제한하면
        
        사용자가 거부감/반발을 느껴 이탈할 수 있음.
        
3. **기존 포모도로/집중 앱과의 차별성 부족**
    - 단순 타이머·통계 수준이면 레드오션.
    - “런처 + NO 플로우 + 자기계발 옵션 묶음”이 USP가 되도록 설계 필요.

### 7.4 가정·의존성

- 사용자는 **하루 1~2개의 자기계발 블록**을 만들 의지가 있음.
- 스마트폰 알람/푸시를 최소한 허용하는 수준의 디지털 리터러시 보유.
- 안드로이드 OS의 알람/딥링크 기능을 활용할 수 있음 (iOS는 좀 더 제약적인 플로우 설계 필요).

## 8. 실험·롤아웃·측정

### 8.1 베타 론칭 계획

- **타깃:**
    - 자기계발 의지가 높고, 디지털 미니멀리즘에 관심 있는 직장인 20–50명
- **기간:**
    - 4주 베타

### 8.2 실험 가설

1. H1: Focus Habit Launcher를 4주 사용한 그룹은
    - 의도된 자기계발 블록 실행률이 **30%p 이상 증가**한다.
2. H2: 런처 기반 선택 UI를 통해
    - “목적 없는 앱 전환” 횟수가 블록당 **2회 이하로 감소**한다.
3. H3: 월간 활동 그래프 제공 시
    - 4주차 Retention(주 3일 이상 사용)이 **50% 이상** 유지된다.

### 8.3 측정 항목
