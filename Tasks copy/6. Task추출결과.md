# MVP 개발 Task 추출 결과 (Integrated Task List)

> **기준 문서:** `5-2.PRD기반 SRS(Software-requirements-specification)_GPT5.md` (SRS v1.1)
> **목적:** AI 에이전트가 수행할 수 있는 수준의 구체적인 개발 Task 정의
> **작성일:** 2025-11-29 (Updated)

---

## 1. 개요 (Overview)

본 문서는 **Focus Habit Launcher**의 MVP 개발을 위해 SRS(요구사항 명세서)를 기반으로 추출된 모든 **개발 Task**를 정의한다. 
SRS v1.1의 모든 기능(Functional) 및 비기능(Non-Functional) 요구사항을 포함하며, 개발 에이전트가 즉시 실행 가능한 단위로 구조화하였다.

### 1.1 Task 통계
- **Total Epics:** 7 (Init & Config, Frontend PoC 포함)
- **Coverage:** SRS v1.1 REQ-FUNC-001 ~ 023 (024 제외/Won't), REQ-NF-001 ~ 008

## 3. 상세 Task 정의 (Detailed Tasks)

각 Task는 `[Task ID] Task Name` 형식을 따르며, 구현에 필요한 상세 요건을 포함한다.

### 3.1 EPIC-0: Initialization & Configuration (INIT_CONFIG)

> **목표:** 안정적인 개발을 위한 기반 환경 및 아키텍처 구축

| Task ID | Task Name | Type | 설명 |
|---|---|---|---|
| **TASK-INIT-001** | **Project Setup** | Config | Android Studio 프로젝트 생성 (MinSDK 26+, Kotlin DSL, Jetpack Compose) |
| **TASK-INIT-002** | **Architecture Setup** | Arch | Hilt(DI), Navigation Component, MVVM 패턴 기본 구조(BaseViewModel 등) 설정 |
| **TASK-INIT-003** | **DB Setup** | Data | Room Database 설정, TypeConverter(Date, List) 유틸리티 구현 |
| **TASK-INIT-004** | **CI Workflow** | DevOps | GitHub Actions: Build & Unit Test 자동화 워크플로우(yml) 작성 |
| **TASK-INIT-005** | **Linter & Format** | DevOps | Ktlint 또는 Detekt 설정으로 코드 스타일 강제 |

## 5. 개발 로드맵 및 우선순위 (Development Roadmap)

### Phase 1: Foundation & Core Logic (Week 1)
- **Focus:** EPIC-0(Init), EPIC-1(DB/Repo), EPIC-2(Alarm Logic)
- **Milestone:** 앱이 빌드되고, 알람이 울리며, DB에 습관 데이터가 저장됨.

### Phase 2: Launcher Flow & UI (Week 2)
- **Focus:** EPIC-3(Launcher UI), EPIC-1(Habit UI)
- **Milestone:** 알람이 울리면 YES/NO 화면이 뜨고 기록을 남길 수 있음.

### Phase 3: Analytics & Export (Week 3)
- **Focus:** EPIC-4(Stats), EPIC-5(Export), EPIC-6(Onboarding)
- **Milestone:** 그래프 확인 가능, CSV 내보내기 가능, 온보딩 적용.

### Phase 4: Polish & QA (Week 4)
- **Focus:** 비기능 요구사항(성능, 안정성), 버그 수정
- **Milestone:** MVP v1.0 출시 준비 완료.
